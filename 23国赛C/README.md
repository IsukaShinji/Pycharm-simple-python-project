# 2023 全国大学生数学建模竞赛 C 题

本目录整理了 2023 年国赛 C 题的代码实现与建模尝试。题目围绕生鲜商超蔬菜类商品的销售规律、补货计划和定价策略展开，核心目标是在历史销售、批发价格、损耗率等数据基础上，给出可解释的销售分析与可执行的经营决策。

这些脚本是我在备战数学建模竞赛期间完成的阶段性实现，重点覆盖数据清洗、销量规律分析、价格-销量关系拟合、补货量与定价优化等环节。代码保留了当时多轮调参和修正的过程，适合作为“从数据分析到优化建模”的完整练习样例阅读。

## 项目内容

- 第一问：分析各蔬菜品类与单品的销量分布、季节性变化、周末效应和大类关联关系。
- 第二问：在历史销售、批发价和损耗率基础上，建立品类层面的需求模型，并给出未来一周的补货与定价策略。
- 第三问：筛选 2023 年 7 月 1 日可售单品，结合需求预测、损耗率、成本和最小陈列约束，使用启发式优化方法寻找单品补货和定价方案。

## 技术路线

1. 使用 `pandas` 读取和清洗附件数据，统一日期、单品编码、销售类型等字段。
2. 按品类、单品、日期、季度等维度聚合销量与价格信息。
3. 使用相关性分析、季节性指标、周末/工作日对比等方法解释销售规律。
4. 使用线性回归、幂函数、Logistic 等模型拟合价格与需求的关系。
5. 在成本、损耗率和需求约束下构造利润函数，通过优化算法搜索补货量与加成率。

| `23国赛C-第二问.py` | 构建品类需求模型，预测未来一周销量并生成补货、定价方案。 |

## 运行说明

脚本依赖比赛附件数据，包括商品信息、销售流水、批发价格和损耗率表。当前代码保留了当时本机的绝对路径，例如 `C:\Users\...\Desktop\23数模C题`，如需复现，需要将路径改为自己的数据目录。

主要依赖：

```bash
pip install pandas numpy matplotlib seaborn scikit-learn scipy openpyxl
```

## 复盘

这个项目的价值不只在最终结果，也在于暴露了数学建模中很重要的一点：先把模型假设、变量关系和约束写清楚，再进入代码实现。第三问的早期版本出现过模型逻辑和实现推进不够同步的问题，后来复盘时也把“先建模、再编码、最后调参”作为竞赛协作中的基本原则。
