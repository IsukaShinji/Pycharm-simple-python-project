# 2025 暑假编程集训

本目录整理 2025 年暑假编程集训期间完成的代码。训练目标是从网页抓取、URL 处理、文本预处理、倒排索引、BM25 检索到 Flask 页面，逐步搭建一个小型站内搜索引擎原型。

这个目录很适合观察一个小工程如何从“单日练习脚本”慢慢演进成“可拆模块的原型”：早期脚本关注单个知识点，后期 `草稿/` 目录把爬虫、预处理、索引、检索和 Web 展示拆成独立文件。

## 系统流程

```mermaid
flowchart LR
    A[目标站点页面] --> B[spider.py / day02<br/>抓取 HTML]
    B --> C[crawled_htm_files<br/>本地 HTML 快照]
    C --> D[preprocessor.py / day03<br/>正文抽取与分词]
    D --> E[inverted_index.py<br/>倒排索引]
    E --> F[retriever.py / day04<br/>关键词检索与排序]
    F --> G[app.py<br/>Flask 搜索页面]
```

## 关键实现

### 网页抓取与 URL 规范化

早期脚本从本地 HTML 解析链接开始，逐步扩展到真实网站抓取。代码中使用：

- `requests` 设置请求头、超时和重试。
- `BeautifulSoup` 解析页面链接和正文。
- `urljoin`、`urlparse`、`url_normalize` 处理相对链接、路径拼接和重复 URL。
- 文件名编码策略保存 HTML，再通过还原函数恢复原始 URL。

这一部分对应 `暑假集训day01.py`、`暑假集训day02.py`、`day2--url处理.py`、`day3--url还原.py` 和 `url处理-还原实验.py`。

### 中文分词与索引

检索部分主要围绕中文页面展开。代码使用 `jieba` 精确分词，并在部分版本中补充 bigram 分词，用于缓解中文短词切分不稳定的问题。

索引结构经历了两个阶段：

- 文本版索引：`day03.py` 和 `暑假集训day03.py` 将词项、文档 ID、词频、URL 映射和文档长度保存到文本文件。
- JSON 模块版索引：`草稿/inverted_index.py` 读取预处理后的 JSON 文档，生成 `inverted_index.json` 和 `vocabulary.json`。

BM25 评分在 `day03.py`、`暑假集训day03.py` 和 `暑假集训day04.py` 中实现，使用常见参数 `k1=1.2`、`b=0.75`，并结合词频、文档长度和文档频率计算相关性。

### Web 检索原型

`草稿/app.py` 是当前更明确的 Web 原型入口。它会自动写入一个简单的 Flask 模板，通过表单接收查询词，并调用 `retriever.basic_retrieval(query)` 返回结果。

需要注意：一级目录中的 `暑假集训day06【app】.py` 当前是空文件，有效 Web 原型不在这个文件里，而在 `草稿/app.py`。

## 运行方式

如果只看模块化原型，推荐从 `草稿/` 目录开始：

```bash
cd 2025编程集训/草稿
python preprocessor.py
python inverted_index.py
python app.py
```

常用依赖：

```bash
pip install requests beautifulsoup4 url-normalize jieba flask
```

运行前需要检查几个路径：

- HTML 输入目录是否与脚本中的 `HTML_DIR` 或类似常量一致。
- `processed/`、`index/` 等输出目录是否存在或能自动创建。
- 停用词文件 `stopwords.txt` 是否存在；不存在时部分脚本会用空集合继续运行。

## 工程复盘

这个项目的意义在于把信息检索的链路完整跑通：抓取页面、清洗文本、建立索引、计算相关性、展示结果。它已经具备小型搜索系统的基本骨架，但距离成熟检索系统仍有差距：

- 爬虫策略较简单，没有 robots、限速队列、断点续跑数据库化等机制。
- 中文分词依赖 `jieba` 和少量自定义词，领域词表仍不完整。
- 排序主要基于 BM25 和词频，缺少标题加权、字段加权、摘要高亮和去重策略。
- Web 页面是功能原型，不是完整前端工程。
- 数据目录保存的是一次抓取快照，线上页面变更后结果不保证一致。

如果继续改进，我会优先把路径和参数集中配置，补充抓取日志，把索引格式从文本/JSON 过渡到更稳定的持久化结构，并加入查询结果摘要与高亮。
